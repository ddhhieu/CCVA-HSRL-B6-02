##  Lý Do Sử Dụng GitHub

Nhóm **B6-02** lựa chọn **GitHub** làm nền tảng quản lý mã nguồn và hợp tác phát triển nhờ các ưu điểm vượt trội sau:

1. **Quản lý phiên bản chặt chẽ (Version Control System):**
   * Giúp theo dõi chi tiết lịch sử thay đổi của từng dòng code (`git commit`), dễ dàng khôi phục (rollback) về trạng thái cũ khi gặp lỗi.
2. **Hợp tác phân nhánh hiệu quả (Branching & Pull Request):**
   * Cho phép các thành viên phát triển từng tính năng riêng biệt trên các nhánh (`feature branch`) độc lập mà không ảnh hưởng đến mã nguồn chính (`main`).
   * Quy trình **Pull Request (PR)** giúp cả nhóm review code chéo trước khi merge, đảm bảo chất lượng lập trình.
3. **Minh bạch và Đánh giá tiến độ (Transparency & Tracking):**
   * Hỗ trợ công cụ **Issues** và **GitHub Projects** để phân chia công việc, giao task và theo dõi tiến độ công việc trực quan của từng thành viên.
4. **Tích hợp tự động hóa (CI/CD):**
   * Dễ dàng kết nối với **GitHub Actions** để tự động kiểm thử (Unit Test), kiểm tra lỗi định dạng và triển khai ứng dụng tự động.
5. **Lưu trữ an toàn & Chia sẻ mã nguồn:**
   * Đảm bảo dữ liệu dự án luôn được sao lưu đám mây an toàn, giúp giảng viên/cố vấn và cộng đồng dễ dàng tiếp cận, đánh giá và đóng góp.
